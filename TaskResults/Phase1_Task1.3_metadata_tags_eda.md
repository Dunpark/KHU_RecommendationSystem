## 4. 결과 해석

### 4.1 핵심 발견

#### 1️⃣ **태그 데이터 품질 우수**
```
태그 보유율: 97.6% (49,628 / 50,872)
games.csv 매칭률: 100%
```
- 거의 모든 게임에 태그 정보 존재
- Content-based 추천에 충분히 활용 가능

#### 2️⃣ **Indie 게임이 과반수 (55%)**
```
Indie 태그: 27,957개 (55.0%)
```
- Steam 플랫폼의 특성: 인디 게임 중심
- Long-tail 게임 대부분이 Indie 게임일 가능성
- → RQ1 분석에서 Indie 게임의 추천 성능 별도 분석 가치

#### 3️⃣ **Singleplayer 중심 (44%)**
```
Singleplayer: 44.4% vs Multiplayer: 12.9%
```
- 싱글플레이 게임이 3.4배 더 많음
- 멀티플레이 게임은 상대적으로 희소 → 태그 기반 세분화 중요

#### 4️⃣ **태그 다양성 충분**
```
유니크 태그: 441개
게임당 평균: 11.5개
16-20개 태그 보유: 39.7%
```
- 풍부한 태그로 세분화된 취향 반영 가능
- → RQ3 (메타데이터 기여도)에서 태그가 핵심 피처일 것으로 예상

### 4.2 연구 질문(RQ) 연결

| RQ | 시사점 |
|----|--------|
| **RQ1** | Indie 게임 55% → Long-tail의 특성 파악 가능 |
| **RQ2** | 태그 기반 Cold-start 보완 가능 (97.6% 커버리지) |
| **RQ3** | 441개 유니크 태그 → **가장 풍부한 Content Feature** |

## 5. 다음 Task에 미치는 영향

### 5.1 태그 피처 엔지니어링 계획

| 방법 | 설명 |
|------|------|
| **Multi-hot Encoding** | 441개 태그 → 441차원 벡터 (희소) |
| **Top-K 태그만 사용** | 상위 50-100개 태그로 축소 |
| **태그 임베딩** | TF-IDF 또는 Word2Vec 스타일 임베딩 |
| **장르 그룹화** | 유사 태그 클러스터링 (예: Action+Shooter) |

### 5.2 태그 없는 게임 처리 (1,244개)

- **방법 1**: 제외 (2.4%이므로 영향 적음)
- **방법 2**: games.csv의 다른 피처로 보완 (rating, price 등)

### 5.3 Content-based 모델 설계 영향

```
Item Vector = [Tag_1, Tag_2, ..., Tag_441] + [rating, price, ...]
```
- 태그 벡터가 Content-based 모델의 핵심
- Multi-hot 인코딩 시 441차원 → 차원 축소 고려
