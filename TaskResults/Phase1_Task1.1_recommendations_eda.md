## 4. 결과 해석

### 4.1 핵심 발견

#### 1️⃣ 긍정 편향 (Positive Bias)
```
추천 vs 비추천 = 84.5% : 15.5% ≈ 5.5 : 1
```
- Steam 리뷰 특성: 불만족한 사용자는 리뷰를 덜 작성하는 경향
- **모델 관점**: `is_recommended=False`가 더 강한 신호일 수 있음

#### 2️⃣ 인기 게임 집중 현상 (Long-tail 증거)
```
100만 행 / 213 게임 ≈ 게임당 4,695 리뷰
```
- 샘플 100만 행에 app_id가 213개밖에 없음
- **극단적인 Long-tail 분포**의 명확한 증거
- → RQ1 (Popular vs Long-tail) 분석의 핵심 근거

#### 3️⃣ 플레이타임 분포 (Right-skewed)
```
평균(201h) >> 중앙값(100h)
```
- Heavy users가 평균을 끌어올림
- 최대값 999.90h → 데이터 캡 가능성
- → RQ2 (Heavy vs Light users) 분석의 근거

#### 4️⃣ 데이터 품질
- 모든 컬럼에서 **결측치 0%** ✅
- 데이터 전처리 부담 최소화

### 4.2 연구 질문(RQ) 연결

| RQ | 시사점 |
|----|--------|
| **RQ1** | app_id 213개가 100만 행 점유 → Zipf's Law 확인 필요 |
| **RQ2** | hours 평균 vs 중앙값 차이 → Heavy/Light 구분 근거 |
| **RQ3** | 긍정 편향 84.5% → hours가 선호 강도로 더 유용할 수 있음 |
